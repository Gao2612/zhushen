# v0.1 UE 5.8 环境下载清单

本清单用于当前电脑尚未安装 UE5 / Android 工具链时的准备工作。
v0.1 固定使用 UE 5.8，不再回退到 5.4 或 5.5。

## 目标版本

- Unreal Engine：5.8
- 项目目录建议：`G:\ZhushenUE5Demo`
- UE 安装目录建议：`G:\UE_5.8`
- Android SDK 建议目录：`G:\Android\Sdk`
- JDK 建议目录：`G:\Program Files\Eclipse Adoptium\jdk-17`

## 必装内容

1. Epic Games Launcher
   - 下载页：https://www.unrealengine.com/download
   - 用途：登录 Epic 账号并安装 Unreal Engine 5.8。

2. Unreal Engine 5.8
   - 安装方式：Epic Games Launcher -> Unreal Engine -> Library -> Engine Versions -> 选择 5.8。
   - 安装完成后确认存在：
     - `G:\UE_5.8\Engine\Binaries\Win64\UnrealEditor.exe`
     - `G:\UE_5.8\Engine\Build\BatchFiles\RunUAT.bat`

3. Visual Studio 2022
   - 下载页：https://visualstudio.microsoft.com/downloads/
   - 必选工作负载：
     - Desktop development with C++
     - Game development with C++
   - 用途：编译 UE C++ 项目与 Windows 包。

4. Android Studio
   - 下载页：https://developer.android.com/studio
   - 用途：提供 Android SDK、platform-tools、build-tools。

5. JDK 17
   - 下载页：https://adoptium.net/temurin/releases/?version=17
   - 用途：Android 打包链路需要 Java。

## UE 5.8 Android 推荐流程

UE 5.8 已强化 Android 工作站自动配置。优先使用 Unreal Editor 内置流程：

1. 打开 `ZhushenActionDemo.uproject`。
2. 进入 `Platforms -> SDK Management -> Android`。
3. 点击 `Install Sdk`。
4. 按提示安装 SDK、NDK、JDK。
5. 关闭命令行和 Unreal Editor，重新登录 Windows。
6. 再次打开 `SDK Management`，确认不再提示安装或修复。

如果自动流程失败，再手动检查：

- `ANDROID_HOME`
- `ANDROID_SDK_ROOT`
- `JAVA_HOME`
- Android SDK Platform 35
- Android build-tools 35.0.0
- Android NDK 25.2.9519653 或 UE 5.8 SDK 管理器推荐版本

## 本仓库脚本

打开下载页：

```powershell
cd E:\zhushen\ue5\ZhushenActionDemo
.\Tools\open-v01-downloads.ps1
```

检查环境：

```powershell
cd E:\zhushen\ue5\ZhushenActionDemo
.\Tools\check-v01-env.ps1
```

打 Windows 包：

```powershell
$env:UE_ROOT = "G:\UE_5.8"
.\Tools\build-v01-windows.ps1
```

打 Android 包：

```powershell
$env:UE_ROOT = "G:\UE_5.8"
.\Tools\build-v01-android.ps1
```

## 验收前不要跳过

- 先确认 `check-v01-env.ps1` 不再显示 `MISSING`。
- 先在 Unreal Editor 里创建或生成 `L_v01_Startup`。
- 再执行 Windows / Android 打包脚本。
- 打包成功后补写 `BuildReports/v0.1-build-report.md`。
