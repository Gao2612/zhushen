# v0.1 C++ Code Structure Review

本审查只针对当前仓库已有 C++ 代码做静态检查，不依赖 UE 5.8 引擎编译。

## 结论

当前 C++ 结构符合 v0.1 “环境验证”目标：代码只负责显示占位体、相机和 HUD，
没有引入战斗、技能、Boss、正式 UI、正式角色美术或复杂特效。

## 模块依赖

- `ZhushenActionDemo.Build.cs` 当前依赖：
  - `Core`
  - `CoreUObject`
  - `Engine`
  - `InputCore`
- 这些依赖对当前 `Pawn`、`HUD`、相机和基础对象足够。
- 当前未使用 Enhanced Input 代码，`.uproject` 启用 `EnhancedInput` 插件但没有造成 v0.1 复杂度扩散。

## Target 配置

- `ZhushenActionDemo.Target.cs` 使用 `TargetType.Game`。
- `ZhushenActionDemoEditor.Target.cs` 使用 `TargetType.Editor`。
- 命名与模块名一致，符合 UE C++ 工程基础要求。

## GameMode / Pawn / HUD

- `AZhushenV01GameMode` 只设置默认 Pawn 和 HUD。
- `AZhushenV01Pawn` 使用引擎内置 Cube 作为占位体，不依赖外部角色资源。
- `AZhushenV01Pawn` 包含 SpringArm 和 Camera，满足 v0.1 俯视观察需求。
- `AZhushenV01Hud` 只绘制标题、环境验证说明和 FPS，符合“非正式 UI”边界。

## 需要等 UE 5.8 编译后确认

- `ConstructorHelpers::FObjectFinder<UStaticMesh>` 引用 `/Engine/BasicShapes/Cube.Cube` 是否在 5.8 中仍可正常解析。
- `GEngine->GetLargeFont()` 与 `GetSmallFont()` 在 Shipping / Android 下是否显示正常。
- Android 横屏下 HUD 文字是否过大或被刘海区域遮挡。

## 后续可选微调

- 如果严格按“胶囊体占位”执行，可在 v0.2 前把 Cube 换成 `UCapsuleComponent`。
- 如果后续加入输入，再决定是否真正使用 Enhanced Input；v0.1 不需要提前接输入。
- 如果 Android HUD 字体过小或过大，再在实机记录后调整 Canvas 坐标和字体。
