## 地图内容

`L_v01_Startup` 由 `Tools/CreateV01StartupMap.py` 生成，包含：

- `V01_GreyboxGround_40m`：40m x 40m 灰盒地面。
- `V01_VisiblePlaceholder_Body`：圆柱体占位身体。
- `V01_VisiblePlaceholder_Head`：球体占位头部。
- `V01_GreyboxScale_North/East/South/West`：四个尺度标记方块。
- `V01_DirectionalLight`：基础方向光。
- `V01_SkyLight`：基础天空光。
- `V01_OverviewCamera`：俯视 45 度附近的场景观察相机。
- `V01_TitleText`：`Zhushen UE5 Demo v0.1` 文字标识。
- `V01_PlayerStart`：默认出生点。

说明：PlayerStart 不是可见角色，因此地图脚本显式生成圆柱 + 球体
占位角色，方便 v0.1 验证时肉眼确认场景内容。

## 当前已知阻塞

1. Windows 正式链路不是编译失败，而是 UAT staging 文件复制失败。
   失败文件为 UE 自带 `dbghelp.dll`。
2. Android 当前电脑不是 NDK 或 C++ 失败，而是 Gradle 依赖解析问题。
3. Android 远端内容版不是 SDK 识别失败，而是 shader cook 牵入过多
   Engine 插件和移动端材质。
4. v0.1 尚未完成最终验收，因为 APK 还没有生成并安装到真机或模拟器。

## 明日继续顺序

1. 修 Windows `dbghelp.dll` staging copy，拿到可重复产出的正式 Shipping 包。
2. 固化 Gradle Maven 镜像或预热 AGP 依赖缓存，拿到 Development APK。
3. 清理 Android cook 牵入的默认插件和 OpenGL ES fallback。
4. 连接 MuMu 或真机，记录横屏、启动时间、包体和帧率。
